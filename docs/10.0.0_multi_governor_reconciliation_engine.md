# Layer 10 — Multi‑Governor Reconciliation Engine

## Architecture Specification

Layer 10 provides deterministic, allocator-free, multi-governor proposal reconciliation for the AILEE Finance Runtime Kernel. This layer resolves diverging or conflicting trading recommendations generated by distinct specialized governors (Compliance, Risk, Liquidity, Return, and Strategy).

### 1. Priority Ladder (GOVERNOR_LADDER_V1)

The priority hierarchy is static and unmutative at runtime, guaranteeing absolute predictability and zero circular dependency:

1. **COMPLIANCE** (Highest priority; absolute veto/hard block authority)
2. **RISK** (Enforces risk-score safety constraints and limits)
3. **LIQUIDITY** (Camps proposal magnitudes based on transactional flow/slippage)
4. **RETURN** (Drives directional refinement with higher yield awareness)
5. **STRATEGY** (Lowest priority base-level asset positioning proposals)

---

### 2. Override Matrix & Deterministic Rules

Overrides and limits are evaluated strictly in ladder order. Higher-priority governors overwrite, clamp, or zero-out proposals made by lower-priority governors:

| Source Governor | Target Governor | Trigger Condition | Override Effect |
|---|---|---|---|
| **COMPLIANCE** | All | `flags` contains `HARD_BLOCK` | Reconciled value is strictly set to `0.0`. |
| **RISK** | RETURN / STRATEGY | `risk_score > 75.0` | Forcefully clamp/limit the proposal magnitude. |
| **LIQUIDITY** | RETURN / STRATEGY | `proposed_value` (as cap) < candidate value | Limit/clamp candidate magnitude to the liquidity limit. |

---

### 3. Canonical Reconciliation Residual Formula

To quantify overall governor disagreement ("tension") for downstream observability and logging, Layer 10 computes a weighted L1 distance:

$$ \text{Residual} = \sum_{i} w_i \cdot |p_i - d| $$

Where:
* $p_i$ is Governor $i$'s proposed value for the asset.
* $d$ is the final reconciled decision value.
* $w_i$ is the static weight configured per governor type:
  * COMPLIANCE: `1.0`
  * RISK: `0.8`
  * LIQUIDITY: `0.6`
  * RETURN: `0.5`
  * STRATEGY: `0.5`

---

### 4. Deterministic Walkthrough Example

Consider an asset with the following conflicting proposals:
* **STRATEGY**: Proposed Value = `100.0`
* **RETURN**: Proposed Value = `120.0`
* **LIQUIDITY**: Limit Proposed Value = `80.0`
* **RISK**: Proposed Value = `50.0` with `risk_score = 80.0` (severe)
* **COMPLIANCE**: Proposed Value = `0.0` with `flags = ReconciliationFlags::NONE` (no block)

#### Stage-by-Stage Traversal:
1. **STRATEGY**: Base value initialized to `100.0`.
2. **RETURN**: Replaces strategy value with `120.0` (direction refinement).
3. **LIQUIDITY**: Candidate magnitude `120.0` exceeds Liquidity limit of `80.0`. Clamps value to `80.0`.
4. **RISK**: Since `risk_score > 75.0` and current candidate `80.0` exceeds Risk proposal of `50.0`, Risk overrides previous stages and clamps the value to `50.0`.
5. **COMPLIANCE**: No hard block flagged. Final decision becomes `50.0`.

#### Residual Calculation:
$$ \text{Residual} = (1.0 \times |0.0 - 50.0|) + (0.8 \times |50.0 - 50.0|) + (0.6 \times |80.0 - 50.0|) + (0.5 \times |120.0 - 50.0|) + (0.5 \times |100.0 - 50.0|) $$
$$ \text{Residual} = 50.0 + 0.0 + 18.0 + 35.0 + 25.0 = 128.0 $$
