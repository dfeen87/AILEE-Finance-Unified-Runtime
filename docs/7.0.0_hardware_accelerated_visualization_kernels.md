# AILLEE 7.0.0: Hardware-Accelerated Visualization Kernels

## Overview
AILLEE 7.0.0 introduces an optional WebGL-based hardware-accelerated visualization kernel within the dashboard plugin. This enhancement significantly increases visual fidelity and rendering efficiency for observability streams by shifting presentation work to the GPU.

## Design Philosophy
Crucially, hardware acceleration is kept **strictly** within the browser/presentation layer. The AILLE C++ engine remains completely unchanged. By designing the WebGL kernel inside the JavaScript client (`webgl_renderer.js`), we maintain:
- Zero dependency changes in the C++ layer.
- Zero allocations, exceptions, or new thread usage in the C++ layer.
- Complete ABI stability of the core engine.

## WebGL Architecture
- **Shader Pipeline:** The WebGL renderer utilizes lightweight point and fragment shaders to dynamically generate colored geometric nodes, data flow particles, and the central core aura.
- **Data-Driven Hardware Shading:** Node properties (size, color, and blending modes) map deterministically to the risk and exposure metrics passed over WebSocket by the observer plugin.
- **Fallback Support:** The JS wrapper (`app.js`) attempts to construct the `WebGLRenderer`. If the browser or hardware doesn't support WebGL, it gracefully falls back to the deterministic Canvas 2D engine introduced in prior versions.

## Visual Upgrades
While preserving the "NASDAQ-style" cinematic aesthetic (deep navy, electric blue, gold, silver), the WebGL kernel supports:
- High-performance blending functions (`gl.BLEND`, `gl.SRC_ALPHA`, `gl.ONE`) for true luminous data flow effects along connection lines.
- Hardware-calculated geometric primitives matching exact asset coordinates on screen.
- Independent rendering pipeline free of DOM update stutters.
