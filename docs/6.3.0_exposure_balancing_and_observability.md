# AILLEE 6.3.0: Exposure Balancing and Observability

## Overview
AILLEE 6.3.0 introduces real-time exposure balancing and observability metrics streaming through the dashboard plugin. To support the growing number of integrated commodities and assets (such as OIL, GOLD, SILVER, COPPER, NATGAS, and PLATINUM), the observer now breaks out the unified "Commodities" aggregation into granular, per-asset exposure values.

## Deterministic Data-Driven Observability
The `LiveAdvisoryObserver` plugin serializes the risk and exposure metrics directly from the core advisory structs into a lightweight JSON payload and streams them over WebSocket.
- **Total Exposure:** The sum of all recommended weights is calculated and streamed as `total_exposure`.
- **Per-Asset Tracking:** The nodes within the presentation layer have been expanded from 4 basic nodes to a comprehensive 9-asset network, pulling strictly from the exact states in `aille_btc.hpp`, `aille_gold.hpp`, etc.

## Dashboard Visualization Updates
The frontend JS visualization (`app.js`) is designed around deterministic principles:
- **No Randomness:** All visual behaviors (size, speed, color) are strictly derived from the incoming data stream and a fixed-step time variable.
- **Exposure Bars:** A new rendering component has been added to display a deterministic exposure bar graph for all assets, rendering `total_exposure` alongside individual asset weights.
- **Animation Scaling:** Flow animation speed maps linearly to the `risk_score` metric of the specific asset, offering immediate visual observability without arbitrary randomness.

## Core Engine Purity
By handling all WebSocket management, JSON formatting, and memory allocation within the `LiveAdvisoryObserver` plugin class, the AILLE C++ engine remains completely allocator-free, exception-free, and ABI-stable. The core framework continues to adhere to strictly predictable cycle counts while the presentation layer handles rich metric visualizations.
