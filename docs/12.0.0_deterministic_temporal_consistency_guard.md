# Layer 12 — Deterministic Temporal Consistency Guard

## Architecture Specification

Layer 12 acts as the final time-domain temporal consistency guardrail for the AILEE Finance Runtime Kernel. It ensures that portfolio decisions, constraints, and governor outputs remain temporally consistent across frames. This prevents silent drift, time-based circular feedback loops, and unexplained jumps in asset allocations.

The system enforces stability and predictability under the version tag `TEMPORAL_CONSISTENCY_V1`.

---

### 1. Deterministic Transition & Consistency Pipeline

Given:
- Previous states $S_t$ (allocations, risk posture, past steps)
- Proposed current states $I_{t+1}$
- Fixed parameters (drift thresholds, dampening factors)

The Layer 12 pipeline applies three sequential operations:

1. **Zero-Drift Expectation Calculation**
   The expected baseline allocation for each asset $\hat{w}_{i, t+1}$ defaults to the previous step's allocation:
   $$ \hat{w}_{i, t+1} = w_{i, t} $$

2. **Oscillation / Circularity Detection & Dampening**
   To prevent feedback loops and flip-flops, the system tracks consecutive step changes:
   $$ \Delta_t = w_{i, t} - w_{i, t-1} $$
   $$ \Delta_{t+1} = w_{i, t+1} - w_{i, t} $$

   An oscillation is detected if the direction of change reverses:
   $$ \text{sign}(\Delta_t) \neq \text{sign}(\Delta_{t+1}) $$
   and both absolute step sizes exceed a small tolerance threshold (e.g., $10^{-6}$ or `0.0001` to avoid noise).

   If an oscillation is detected for asset $i$:
   - The proposed current allocation is dampened to half of its planned step:
     $$ w_{i, t+1} \leftarrow w_{i, t} + 0.5 \cdot \Delta_{t+1} $$
   - The `oscillation_detected` flag is set in the asset's `TemporalState`.

3. **Temporal Drift Clamping**
   The deviation of the final (potentially pre-dampened) allocation from the zero-drift expectation is calculated:
   $$ \text{drift} = |w_{i, t+1} - \hat{w}_{i, t+1}| $$

   If $\text{drift}$ exceeds the configured `max_drift_threshold` (e.g., `0.05`):
   - The allocation change is capped relative to the expectation:
     $$ w_{i, t+1} \leftarrow \hat{w}_{i, t+1} + \text{sign}(w_{i, t+1} - \hat{w}_{i, t+1}) \times \text{max\_drift\_threshold} $$

---

### 2. Canonical Temporal Residual

The total magnitude of the temporal adjustment across all assets is quantified via the **Canonical Temporal Residual**:
$$ \text{TemporalResidual} = \sum_i |w_{i, t+1} - \hat{w}_{i, t+1}| $$

This metric is summed and captured at the portfolio level in `TemporalPortfolioState` as `residual_sum`.

---

### 3. Struct Definitions (64-byte Cache-Aligned)

All Layer 12 data structures are designed to be exactly 64 bytes with explicit padding and alignment:

```cpp
struct alignas(64) TemporalState {
    AssetId asset_id;                 // 2 bytes
    uint8_t flags;                    // 1 byte (bit 0: oscillation_detected)
    uint8_t _padding1[5];             // 5 bytes alignment
    double prev_allocation;           // 8 bytes (w_{i,t})
    double prev_risk_score;           // 8 bytes (risk_{i,t})
    uint8_t reserved[40];             // 40 bytes padding -> 64 bytes total
};

struct alignas(64) TemporalResidual {
    AssetId asset_id;                 // 2 bytes
    uint8_t _padding1[6];             // 6 bytes alignment
    double expected_allocation;       // 8 bytes (\hat{w}_{i,t+1})
    double actual_allocation;         // 8 bytes (w_{i,t+1})
    double residual;                  // 8 bytes (|w_{i,t+1} - \hat{w}_{i,t+1}|)
    uint8_t reserved[32];             // 32 bytes padding -> 64 bytes total
};

struct alignas(64) TemporalTraceStep {
    AssetId asset_id;                 // 2 bytes
    uint8_t action_taken;             // 1 byte (0: NO_CHANGE, 1: CLAMPED, 2: DAMPENED, 3: OSCILLATED_AND_CLAMPED)
    uint8_t _padding1[5];             // 5 bytes alignment
    double before_value;              // 8 bytes (original proposed w_{i,t+1})
    double after_value;               // 8 bytes (final resolved w_{i,t+1})
    double residual;                  // 8 bytes
    char log[32];                     // 32 bytes compact log string -> 64 bytes total
};

struct alignas(64) TemporalPortfolioState {
    double portfolio_risk;            // 8 bytes
    double residual_sum;              // 8 bytes (Canonical Temporal Residual sum)
    uint8_t reserved[48];             // 48 bytes padding -> 64 bytes total
};
```

### 4. Walkthrough Example

Consider an asset BTC (AssetId = 1) experiencing temporal oscillation:

1. **History**:
   - Time $t-1$: Allocation $w_{\text{BTC}, t-1} = 0.20$
   - Time $t$: Allocation $w_{\text{BTC}, t} = 0.35$
   - This means $\Delta_t = 0.35 - 0.20 = +0.15$.

2. **Time $t+1$ Proposal**:
   - Proposed $w_{\text{BTC}, t+1} = 0.15$.
   - This means $\Delta_{t+1} = 0.15 - 0.35 = -0.20$.

3. **Layer 12 Analysis**:
   - Expected allocation: $\hat{w}_{\text{BTC}, t+1} = 0.35$.
   - Sign check: $\text{sign}(\Delta_t) = +1 \neq \text{sign}(\Delta_{t+1}) = -1$.
   - **Oscillation detected!**
   - Apply oscillation dampening:
     $$ w_{\text{BTC}, t+1} \leftarrow 0.35 + 0.5 \times (-0.20) = 0.25 $$

   - Now, check temporal drift limits on the dampened value ($0.25$):
     - Expectation: $0.35$. Dampened allocation: $0.25$.
     - $\text{drift} = |0.25 - 0.35| = 0.10$.
     - Max drift threshold: $0.05$.
     - $\text{drift} > 0.05$, so apply drift clamp:
       $$ w_{\text{BTC}, t+1} \leftarrow 0.35 - 0.05 = 0.30 $$

   - Final allocation after Layer 12: $0.30$.
   - All temporal transitions are safe, smooth, and robust.
