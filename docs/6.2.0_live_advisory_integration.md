# AILLEE 6.2.0 — Live Advisory Integration

This document outlines the architecture and integration steps for the AILLEE 6.2.0 Live Advisory Dashboard, which connects the `IAnalyticsObserver` interface to the HTML/JS web dashboard using a deterministic, lightweight WebSocket stream.

## Architecture

The integration utilizes the AILLE Engine's plugin system to maintain the core engine's strict memory constraints (allocator-free, zero-overhead, ABI-stable).

1. **`LiveAdvisoryObserver` (C++ Plugin)**:
   - Inherits from `AILLE::Plugins::IAnalyticsObserver`.
   - Initialized with pointers to the individual advisory structs (e.g., `BTCAdvisory`, `ETHAdvisory`, `OILAdvisory`, `MacroSignalAdvisory`).
   - Runs an embedded `websocketpp` (with standalone ASIO) server on a background thread.
   - Upon every engine cycle (`onDecisionRouted()`), it safely reads the advisory structs, constructs a JSON payload, and broadcasts it to connected WebSocket clients.
   - Preserves engine purity: all networking, dynamic memory allocation (for JSON strings), and threading occur entirely within the plugin context, isolated from the determinism of `AILLEEngine`.

2. **Dashboard Frontend (`ailee_plugins/plugins/dashboard/`)**:
   - An HTML5 Canvas visualization.
   - Connects to the `LiveAdvisoryObserver` WebSocket (`ws://localhost:9002`).
   - Dynamically updates nodes and rendering (size, speed, color) based on real-time risk scores and recommended weights from the engine's multi-asset advisory output.

## Dependencies

The `LiveAdvisoryObserver` requires `websocketpp` and standalone `asio`. These are header-only libraries.
To download them to the `external/` directory, run:

```bash
./setup_websocket.sh
```

## Running the Dashboard Feed

A demonstration server is provided in `examples/dashboard_server.cpp`. It instantiates the AILLE Engine, creates state objects for all assets, and attaches the `LiveAdvisoryObserver`.

To build and run:

```bash
# 1. Download dependencies (if not already done)
./setup_websocket.sh

# 2. Build the dashboard server
make dashboard_server

# 3. Run the server
./dashboard_server
```

Once the server is running, simply open `ailee_plugins/plugins/dashboard/index.html` in any modern web browser to view the live, deterministic flow of the AILLEE 6.2.0 Engine in real-time.
